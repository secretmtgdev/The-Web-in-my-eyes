Accessing a Webpage, sparknotes edition

There is a great article by Maneesha Wijesinghe @ Medium.com, please check it out for more depth on the subject matter. This page is my sparknote version from her article.

The plan

Not a whole lot here other than knowing the page you want to access.

Finding the IP address

  • If found in Browser cache, return IP adddress
  • Else if found in OS cache, return IP address
  • Else if found in Router cache, return IP address
  • Else if found in ISP cache, return IP address
  • Else use DNS to query and find IP address
    • Access '.' aka root server
    • Access '.com' server
    • Access 'site.com' server, if target || contains subdomain queried, return address, else return error

The handshake

  • Client sends message to server
  • Server either accepts or rejects and sends decision back to client
  • Client gives a thumbs up to the response

The actual request

Client sends HTTP request to server using the appropriate HTTP method

The decision

  • Server receives and reads request
  • Server attempts to find sources
  • Server constructs message to send back

The final act

  • Display raw HTML
  • Send GET request to access CSS, images, JS sources

Response types

  • 1xx - Information
  • 2xx - Success
  • 3xx - Redirection
  • 4xx - Client error
  • 5xx - Server error
